# Onboarding Paths

How a new hire (or a newly activated [agent](../agents/README.md)) gets productive fast. Owned by
[People & HR](../hr/README.md) via the [Onboarding Agent](../agents/hr/onboarding-agent.md); every role
has a **day 1 → day 30** path. This page holds the shared arc plus per-track specifics.

## The shared arc (every role)

| Phase | Goal | Key steps |
|---|---|---|
| **Before day 1** | Ready to work | Access provisioned (least privilege, [security](../security/README.md)), equipment, buddy assigned |
| **Day 1** | Oriented | Read [company identity](../company/README.md), [principles](../company/principles.md), and [how we work](../governance/README.md); meet the team |
| **Week 1** | First contribution | Ship one small, real thing; learn the [department](../STRUCTURE.md) and its [standards](../standards/README.md) |
| **Weeks 2-4** | Owning work | Take a scoped piece end-to-end; join the [cadence](../operations/examples/cadence-example.md); first feedback |
| **Day 30** | Ramped | Independently delivering to the [Definition of Done](../governance/definition-of-done.md); check-in |

## Per-track first-30-day focus

| Track | First real win | Learn deeply |
|---|---|---|
| [Engineering](../engineering/README.md) | Ship a reviewed PR to production | [standards](../engineering/standards.md), [testing](../engineering/testing-standards.md), [review](../engineering/review-process.md) |
| [Product](../product/README.md) | Write a one-pager or groom the backlog | [discovery](../product/discovery.md), [prioritization](../product/decision-frameworks.md) |
| [Design](../design/README.md) | Ship an on-system, accessible screen | [design system](../design/design-system.md), [accessibility](../design/accessibility.md) |
| [AI](../ai/README.md) | Ship a prompt/model change through the eval gate | [evaluation](../ai/evaluation-and-testing.md), [prompt memory](../memory/prompt-memory.md) |
| [Sales](../sales/README.md) | Run a real discovery call | [GTM](../strategy/go-to-market.md), the [deal-handoff](../sales/checklists/deal-handoff-checklist.md) |
| [Customer Success](../customer-success/README.md) | Onboard a customer to first value | [health scoring](../customer-success/README.md), [support](../post-launch/support.md) |
| Any [department](../company/structure.md) | The department's smallest real deliverable | its `README.md`, `standards.md`, and `workflows.md` |

## Principle

Onboarding is a product: it has an owner, a metric ([time to first contribution](../hr/metrics.md)), and it
improves every cycle. See the [Company-Building Guide](company-building-guide.md) for the skills each track builds.
