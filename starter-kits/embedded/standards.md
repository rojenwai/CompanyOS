# Embedded Systems - Added Standards

These extend the core [standards](../../standards/README.md) with what this company type specifically requires.

1. Firmware is versioned, tested (including on hardware-in-the-loop), and resource-budgeted.
2. OTA updates are signed, atomic, and recoverable - never brick a device.
3. Secure boot and encrypted storage where the threat model requires it ([security](../../security/README.md)).
4. Reliability is validated against real environmental conditions ([hardware](../../hardware/README.md)).
