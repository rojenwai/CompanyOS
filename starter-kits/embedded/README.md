# Starter Kit: Embedded Systems

A company building firmware-driven products on constrained hardware (devices, controllers, appliances).

## What it inherits (unchanged)

- The full [company identity](../../handbook/company/), [governance](../../handbook/governance/), and [kernel](../../ai/orchestration/) ([memory](../../ai/memory/), [agents](../../ai/agents/)).
- The [engineering](../../handbook/departments/engineering/), [product](../../handbook/departments/product/), and [post-launch](../../handbook/departments/post-launch/) departments and every other core department.
- The [SDLC](../../handbook/workflows/sdlc.md), [standards](../../handbook/standards/), [templates](../../handbook/templates/), and [strategy](../../handbook/departments/strategy/).

## What it adds / emphasizes

| Delta | Where |
|---|---|
| Domain agents & activated departments | [agents.md](agents.md) |
| Domain standards | [standards.md](standards.md) |
| Domain delivery lifecycle | [workflows.md](workflows.md) |
| How to stand this up | [getting-started.md](getting-started.md) |

## Why these deltas

Embedded products run on constrained resources with slow, costly iteration. This kit makes firmware rigor, resource discipline, reliability, and safe OTA first-class.

## Swapping this kit

To build a different company type, copy [starter-kits/ai-startup/](../ai-startup/) or this folder and replace
the domain deltas with your own - the inheritance model is identical.
