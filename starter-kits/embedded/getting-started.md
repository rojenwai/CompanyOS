# Embedded Systems - Getting Started

1. Adopt the core; define resource and reliability budgets up front.
2. Activate [hardware](../../handbook/departments/hardware/README.md); build firmware with HIL testing.
3. Design signed, atomic, recoverable OTA from the start.
4. Validate against environmental conditions ([Hardware Test Agent](../../ai/agents/hardware/hardware-test-agent.md)).
5. Pilot, then move to production with monitoring.
6. Maintain firmware via [post-launch](../../handbook/departments/post-launch/maintenance.md).

> Everything not listed here comes from core Company OS - this kit only adds the domain-specific deltas.
