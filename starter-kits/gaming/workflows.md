# Gaming - Added Workflow

Layered on top of the core [SDLC](../../handbook/workflows/sdlc.md), this domain adds:

```
Concept -> prototype (find the fun) [gate]
  -> vertical slice (prove quality bar)
  -> production (content at scale)
  -> soft launch (measure D1/D7/D30 retention) [gate]
  -> global launch -> live-ops (events, seasons, balance)
  -> sunset with player communication
```

## How it maps to core

- Game feel and playtesting use [design](../../handbook/departments/design/README.md).
- Retention cohorts and balance use [data](../../handbook/departments/data/README.md) and [product analytics](../../handbook/departments/product/analytics.md).
- Live-ops, incidents, and rollback use [post-launch](../../handbook/departments/post-launch/README.md).
- Community and launch use [marketing](../../handbook/departments/marketing/README.md); sunset uses [end-of-life](../../handbook/departments/post-launch/end-of-life.md).
