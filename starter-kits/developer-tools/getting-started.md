# Developer Tools - Getting Started

1. Adopt the core and validate the developer problem.
2. Design the API contract-first ([API standards](../../engineering/api-standards.md)).
3. Stand up [documentation](../../documentation/README.md) with generated reference and a quickstart.
4. Ship an SDK/CLI; measure time-to-first-successful-call.
5. Build community feedback loops into [product feature requests](../../post-launch/feature-requests.md).
6. Commit to versioning and deprecation discipline from v1.

> Everything not listed here comes from core Company OS - this kit only adds the domain-specific deltas.
