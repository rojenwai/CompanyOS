# Developer Tools - Added Standards

These extend the core [standards](../../standards/README.md) with what this company type specifically requires.

1. Every public API is documented with working examples before release ([documentation](../../documentation/README.md)).
2. No breaking API change without a version bump and migration path ([deprecation policy](../../post-launch/deprecation-policy.md)).
3. Time-to-first-successful-call is a tracked north-star for onboarding.
4. Reference docs are generated from the source of truth so they cannot drift.
