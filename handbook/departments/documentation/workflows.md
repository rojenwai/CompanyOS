# Documentation - Workflows

The core repeatable processes this department runs.

### Document a feature

1. Get the spec and reader task
2. Write task-oriented, example-first docs
3. Verify accuracy against the product
4. Publish and structure
5. Add to the freshness register

### Keep docs current

1. Detect drift ([doc drift check](../post-launch/automation.md))
2. Identify the changed behavior
3. Update or retire the doc
4. Verify accuracy
5. Reset the freshness date
