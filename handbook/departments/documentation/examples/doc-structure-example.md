# Example - Documentation Information Architecture

> Worked example - illustrative, not a live record.

```
docs/
|-- get-started/     (task: first success in 10 minutes)
|-- guides/          (task: accomplish a specific goal)
|-- concepts/        (understand how it works)
|-- reference/       (API + config, generated from source)
|-- troubleshooting/ (fix a problem)
`-- changelog/       (what changed, per release)
```

**Rule:** each page states the reader's task at the top; reference is generated from the spec so it cannot drift from the API.
