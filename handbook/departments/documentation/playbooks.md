# Documentation - Playbooks

Situational guides. Each names the trigger and the steps.

### A doc is wrong

1. Confirm the current correct behavior
2. Fix or remove the doc immediately
3. Check for related duplicates
4. Verify the fix
5. Add drift detection if systemic

### Feature shipped without docs

1. Block the [documentation gate](quality-gates.md)
2. Draft the docs from the spec
3. Verify against the product
4. Publish
5. Reinforce docs-as-part-of-done
