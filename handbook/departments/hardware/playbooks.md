# Hardware - Playbooks

Situational guides. Each names the trigger and the steps.

### A field defect appears

1. Assess safety impact first
2. Contain (pause shipments / disable feature if needed)
3. Root-cause with test
4. Fix via firmware OTA or hardware revision
5. Manage RMA with [post-launch](../post-launch/README.md)

### A component goes end-of-life

1. Assess inventory and exposure
2. Qualify an alternative
3. Validate the change by test
4. Update BOM and docs
5. Roll into the next build
