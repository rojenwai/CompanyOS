# Design - Workflows

The core repeatable processes this department runs.

### Design a feature

1. Understand the problem and requirements from [product](../product/requirements.md)
2. Research users and mental models ([ux-research.md](ux-research.md))
3. Design flows and information architecture
4. Design on-system UI with every state
5. Prototype and [usability-test](usability-testing.md)
6. Accessibility audit
7. Hand off specs and tokens to [engineering](../engineering/README.md)

### Evolve the design system

1. Identify a recurring pattern or gap
2. Design the component accessible by default
3. Version and document it
4. Publish and drive adoption
5. Retire the one-off variants
