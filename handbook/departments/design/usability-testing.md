# Usability Testing

Validate designs with real users before and after building.

## Evaluate

Task completion · time on task · error rate · navigation · user satisfaction · accessibility.

## Process

1. Define tasks that map to the [customer journey](../product/customer-journey.md).
2. Observe representative users attempting them.
3. Record friction, errors, and quotes.
4. Prioritize fixes and feed them into [prioritization](../product/decision-frameworks.md).

Results are stored in [research memory](../../../ai/memory/research-memory.md) and inform the next
[iteration](../post-launch/README.md).
