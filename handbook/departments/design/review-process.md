# Design - Review Process

Design is reviewed for system compliance (does it use the design system?), accessibility (does it meet WCAG?), and clarity (validated with users where significant). Design review is a [quality gate](quality-gates.md) before engineering builds.

Reviews are independent - the [Reviewer](../../../ai/orchestration/reviewer.md) never assumes the author is
correct. Security-sensitive work also requires the
[Security Reviewer](../../../ai/orchestration/security-reviewer.md).
