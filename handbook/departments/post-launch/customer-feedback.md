# Customer Feedback

Systematically collect and route what customers tell us.

## Sources

Support tickets · reviews · analytics · interviews · surveys · feature requests · bug reports ·
usage metrics.

## Process

1. Collect from all channels.
2. Classify (bug / request / praise / churn signal) and deduplicate.
3. Route: bugs → [bug-management](bug-management.md); requests → [feature-requests](feature-requests.md);
   themes → [product](../product/README.md).
4. Quantify against [analytics](analytics.md) to separate loud voices from common needs.

## Use

Feedback drives the [continuous improvement loop](continuous-improvement.md) and
[product evolution](product-evolution.md). It is stored in
[customer memory](../../../ai/memory/customer-memory.md) and, where it informs strategy,
[research memory](../../../ai/memory/research-memory.md).
