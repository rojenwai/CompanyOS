# Product SOPs

## SOP-01: Write a PRD
1. Confirm the problem is validated ([discovery](discovery.md)). 2. Copy the [PRD template](../../templates/documents/prd.md). 3. Fill goals, non-goals, requirements, success metrics. 4. Circulate for [review](review-process.md).

## SOP-02: Groom the backlog
1. Ensure each item has description, estimate, acceptance criteria, owner. 2. Re-rank via a [framework](decision-frameworks.md). 3. Keep the top of the backlog [Ready](../engineering/definition-of-ready.md).

## SOP-03: Plan a release
1. Select scope with MoSCoW. 2. Confirm dependencies. 3. Set the release window. 4. Prepare the [launch checklist](quality-gates.md).

## SOP-04: Run an experiment
Follow [analytics.md](analytics.md): hypothesis → variants → metric → decision rule → record result.
