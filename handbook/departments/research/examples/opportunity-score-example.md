# Example - Opportunity Score

> Worked example - illustrative, not a live record.

**Opportunity:** Automated incident postmortems for mid-size SaaS teams.

| Criterion | Score (1-10) | Rationale |
|---|---|---|
| Problem severity | 8 | Teams repeatedly relive the same outages; measurable cost |
| Frequency | 7 | Every incident; multiple per month for the segment |
| Urgency | 6 | Painful but not existential |
| Technical feasibility | 7 | Achievable with current LLM + observability tooling |
| Defensibility | 6 | Data network effects if we aggregate anonymized learnings |
| Market size | 7 | Large SaaS mid-market; clear willingness to pay |

**Weighted score:** 6.9 / 10 -> **Pursue.** Recommended to the Innovation Review Board with a cheap validation test (5 design-partner interviews) as the next step.
