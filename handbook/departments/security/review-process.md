# Security - Review Process

Security review is independent and can block. It checks threat coverage, control implementation, secret handling, and scan results. Critical and high findings block the release; the [Security Reviewer](../../../ai/orchestration/security-reviewer.md) block cannot be auto-overridden.

Reviews are independent - the [Reviewer](../../../ai/orchestration/reviewer.md) never assumes the author is
correct. Security-sensitive work also requires the
[Security Reviewer](../../../ai/orchestration/security-reviewer.md).
