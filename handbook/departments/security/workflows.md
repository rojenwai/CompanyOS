# Security - Workflows

The core repeatable processes this department runs.

### Secure a new feature

1. Threat-model the design
2. Specify controls
3. Review code for security
4. Scan dependencies and run SAST/DAST
5. Pass the [Security Reviewer](../../../ai/orchestration/security-reviewer.md) gate
6. Monitor in production

### Respond to a vulnerability

1. Ingest and triage the finding
2. Rate severity and exploitability
3. Assign owner and SLA
4. Patch and verify
5. Record and learn
