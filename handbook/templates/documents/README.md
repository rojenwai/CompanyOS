# Document Templates

Company-wide, fill-in-the-blank documents. Copy the one you need into your working folder.

| Template | Use when |
|---|---|
| [prd.md](prd.md) | Defining a product or major feature |
| [rfc.md](rfc.md) | Proposing a significant technical or process change |
| [adr.md](adr.md) | Recording an architecture decision |
| [incident-report.md](incident-report.md) | Documenting an incident and its postmortem |
| [research-report.md](research-report.md) | Reporting a research investigation |
| [security-review.md](security-review.md) | Reviewing the security of a change or system |
| [release-checklist.md](release-checklist.md) | Shipping a release |
| [meeting-notes.md](meeting-notes.md) | Capturing decisions and action items from a meeting |
| [status-report.md](status-report.md) | Weekly / monthly progress reporting |
| [okr.md](okr.md) | Setting quarterly Objectives and Key Results |
| [one-pager.md](one-pager.md) | Aligning on an idea before a full PRD or RFC |
| [offer-letter.md](offer-letter.md) | Extending a hiring offer |
| [employment-agreement.md](employment-agreement.md) | The binding employment contract (structure only) |
| [board-update.md](board-update.md) | The recurring board / investor update |
| [sales-proposal.md](sales-proposal.md) | Proposing to a customer, anchored on their outcome |
| [qbr.md](qbr.md) | The quarterly business review with a customer |
| [dpa.md](dpa.md) | Data processing agreement for a vendor or customer |

All templates follow the [universal output format](../../governance/universal-output-format.md) where applicable.

> ⚠️ The legal templates ([employment-agreement.md](employment-agreement.md), [dpa.md](dpa.md)) are
> **structural checklists, not legal advice**. Counsel must review them, and signature requires
> [human approval](../../../ai/orchestration/approval-engine.md).
