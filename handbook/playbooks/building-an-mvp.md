# Playbook: Building an MVP

**When to use:** you have a validated problem and need to ship the first version.

1. **Confirm the problem is validated** — [discovery](../departments/product/discovery.md) evidence and a target
   [persona](../departments/product/user-personas.md) exist. If not, stop and validate first.
2. **Find the one critical job** — the single problem the MVP must solve exceptionally well
   ([MVP planning](../../ai/agents/product/mvp-planning-agent.md)).
3. **Scope ruthlessly** — include only what's needed for that job; defer everything else explicitly
   (MoSCoW, [prioritization](../departments/product/decision-frameworks.md)).
4. **Define the proof metric** — the [success metric](../departments/product/metrics.md) that shows the problem is solved.
5. **Write it up** — a lean [PRD](../templates/documents/prd.md) with goals, non-goals, and metrics.
6. **Build to Ready** — stories that meet the [Definition of Ready](../departments/engineering/definition-of-ready.md);
   hand to [engineering](../departments/engineering/README.md).
7. **Ship and learn** — [launch](launching-a-product.md), measure the proof metric, iterate.

**Success criteria:** the smallest **useful** product ships and validates (or invalidates) the core problem.
**Escalate if:** scope keeps expanding without evidence → the [no-PMF playbook](../departments/product/playbooks.md).
