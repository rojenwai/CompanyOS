# Onboarding Paths

How a new hire (or a newly activated [agent](../../ai/agents/README.md)) gets productive fast. Owned by
[People & HR](../departments/hr/README.md) via the [Onboarding Agent](../../ai/agents/hr/onboarding-agent.md); every role
has a **day 1 → day 30** path. This page holds the shared arc plus per-track specifics.

## The shared arc (every role)

| Phase | Goal | Key steps |
|---|---|---|
| **Before day 1** | Ready to work | Access provisioned (least privilege, [security](../departments/security/README.md)), equipment, buddy assigned |
| **Day 1** | Oriented | Read [company identity](../company/README.md), [principles](../company/principles.md), and [how we work](../governance/README.md); meet the team |
| **Week 1** | First contribution | Ship one small, real thing; learn the [department](../../STRUCTURE.md) and its [standards](../standards/README.md) |
| **Weeks 2-4** | Owning work | Take a scoped piece end-to-end; join the [cadence](../departments/operations/examples/cadence-example.md); first feedback |
| **Day 30** | Ramped | Independently delivering to the [Definition of Done](../governance/definition-of-done.md); check-in |

## Per-track first-30-day focus

| Track | First real win | Learn deeply |
|---|---|---|
| [Engineering](../departments/engineering/README.md) | Ship a reviewed PR to production | [standards](../departments/engineering/standards.md), [testing](../departments/engineering/testing-standards.md), [review](../departments/engineering/review-process.md) |
| [Product](../departments/product/README.md) | Write a one-pager or groom the backlog | [discovery](../departments/product/discovery.md), [prioritization](../departments/product/decision-frameworks.md) |
| [Design](../departments/design/README.md) | Ship an on-system, accessible screen | [design system](../departments/design/design-system.md), [accessibility](../departments/design/accessibility.md) |
| [AI](../departments/ai-engineering/README.md) | Ship a prompt/model change through the eval gate | [evaluation](../departments/ai-engineering/evaluation-and-testing.md), [prompt memory](../../ai/memory/prompt-memory.md) |
| [Sales](../departments/sales/README.md) | Run a real discovery call | [GTM](../departments/strategy/go-to-market.md), the [deal-handoff](../departments/sales/checklists/deal-handoff-checklist.md) |
| [Customer Success](../departments/customer-success/README.md) | Onboard a customer to first value | [health scoring](../departments/customer-success/README.md), [support](../departments/post-launch/support.md) |
| Any [department](../company/structure.md) | The department's smallest real deliverable | its `README.md`, `standards.md`, and `workflows.md` |

## Principle

Onboarding is a product: it has an owner, a metric ([time to first contribution](../departments/hr/metrics.md)), and it
improves every cycle. See the [Company-Building Guide](company-building-guide.md) for the skills each track builds.
