# Hardware - Standard Operating Procedures

### SOP-01 - Release a hardware revision

1. Freeze the design after review
2. Pass the full test plan
3. Confirm manufacturability
4. Pilot build and validate
5. Update BOM and docs
6. Release with a version

### SOP-02 - Deploy an OTA update

1. Pass firmware tests and HIL
2. Canary with rollback
3. Verify field health
4. Progressive rollout
5. Monitor and support recovery
