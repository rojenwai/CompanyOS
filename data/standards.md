# Data - Standards

Testable standards, not aspirations. These extend the company-wide
[standards](../standards/README.md).

1. Every metric has one definition, an owner, and a catalog entry.
2. Pipelines are code, versioned, tested, and monitored.
3. Data-quality tests (freshness, completeness, validity) gate publication.
4. Sensitive data is classified, minimized, and least-privilege.
5. Analyses state method and uncertainty; no p-hacking.
