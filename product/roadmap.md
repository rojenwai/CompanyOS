# Roadmap, Backlog & Release Planning

## Product roadmap

Roadmaps communicate **direction, not promises**. Include: vision · objectives · milestones ·
dependencies · release windows · success metrics. Organize by outcome, not feature list.

## Product backlog

Each backlog item includes: description · priority · estimate · dependencies · acceptance criteria ·
owner · status. Ordered by [prioritization](decision-frameworks.md); the top is always
[Ready](../engineering/definition-of-ready.md).

## Release planning

Plan across release stages:

Alpha → Beta → Release Candidate → General Availability → Maintenance → End-of-Life.

Each release passes the [launch checklist](quality-gates.md) and follows the
[release checklist](../templates/documents/release-checklist.md). Release types and version lifecycle
are owned jointly with [post-launch/release-management](../post-launch/release-management.md).
