# Pull Request Checklist

Before requesting review:

- [ ] Branch follows the [naming convention](../workflows.md)
- [ ] Commits follow [Conventional Commits](../workflows.md)
- [ ] Task met the [Definition of Ready](../definition-of-ready.md)
- [ ] Tests added for every acceptance criterion; coverage targets met
- [ ] No secrets, credentials, or personal data in code or logs
- [ ] Public APIs documented; user-facing docs updated
- [ ] Performance impact considered/measured
- [ ] PR description includes: Summary · Problem · Solution · Testing · Screenshots (if UI) · Performance Impact · Security Impact · Breaking Changes

Before merge:

- [ ] CI green
- [ ] Required reviews obtained (2, or 3 for critical infrastructure)
- [ ] Security and documentation sign-off where applicable
- [ ] [Definition of Done](../quality-gates.md) satisfied
