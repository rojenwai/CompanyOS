# Documentation - Mission

**Make the right knowledge easy to find and trust.**

Undocumented software is unfinished software, and stale docs are worse than none. Documentation ensures every product and process is explained accurately, structured well, and kept current.

Priority order when they conflict:

1. **Accuracy** - docs match reality; stale docs are bugs.
2. **Findability** - the reader reaches the answer fast.
3. **Clarity** - written for the reader's context and level.
4. **Currency** - docs update with the thing they describe.
