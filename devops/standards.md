# DevOps - Standards

Testable standards, not aspirations. These extend the company-wide
[standards](../standards/README.md).

1. All infrastructure and pipelines are code, versioned and reviewed.
2. No deploy without automated tests, security gates, and a tested rollback.
3. Every environment is reproducible from source.
4. Secrets live in a vault; never in code, images, or logs.
5. Every deploy is observable and recorded.
